## 改进建议

1. **Pyright 配置文件位置**
   - 将 `pyrightconfig.json` 移至项目根目录
   - 修改 `include: ["backend"]` 以匹配目录结构
   - 关联问题: pyrightconfig-location

2. **泛型类型参数命名**
   - 使用公共命名 `T` 而非私有命名 `_T`
   - 避免触发 Ruff UP049 警告
   - 关联问题: generic-type-parameter-naming

3. **Repository 方法命名一致性**
   - 重构后全局搜索方法调用
   - 统一方法命名规范（如 `_by_id` 后缀）
   - 依赖 Pyright 类型检查发现不一致
   - 关联问题: repository-method-naming

4. **Decimal 类型转换**
   - 使用 `# type: ignore[assignment]` 绕过 SQLAlchemy 类型存根限制
   - 精确定位忽略范围，不影响其他检查
   - 关联问题: decimal-type-conversion

5. **asyncio.run_in_executor 类型兼容**
   - 使用 `# type: ignore[arg-type]` 绕过多进程类型限制
   - 添加注释说明运行时兼容性
   - 关联问题: asyncio-run_in_executor-compat
