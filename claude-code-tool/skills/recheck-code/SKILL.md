# Recheck Code

对最近修改的代码进行全面的质量检查。

## 触发条件

用户说以下任一短语时触发：
- "recheck-code"
- "检查代码"
- "代码质量检查"
- "recheck"
- "/recheck-code"

## 检查清单

执行以下检查项目：

### 1. Bug 检查
- 代码逻辑是否正确？
- 边界条件是否处理？
- 异常处理是否完善？
- 类型错误可能性？

### 2. 重复代码检查
- 是否存在可以抽取的重复逻辑？
- 是否有可以合并的相似代码块？
- 是否有未使用的导入或变量？

### 3. 遗漏检查
- 是否有 TODO/FIXME 未处理？
- 是否有 import 语句多余或缺失？
- 是否有未使用的变量或函数？
- 相关的测试文件是否需要更新？

### 4. 一致性检查
- 变量命名是否一致？
- 代码风格是否符合项目规范？
- 是否有复制粘贴留下的痕迹（变量名不匹配等）？

## 执行步骤

### 步骤 1: 识别修改范围

使用 git diff 查看最近修改的文件：

```bash
# 查看未暂存的修改
git diff --name-only

# 查看已暂存的修改
git diff --cached --name-only

# 查看最近 N 次提交的修改
git diff HEAD~N --name-only
```

### 步骤 2: 读取修改内容

使用 Read 工具读取每个修改的文件，重点关注：
- 实际修改的代码部分
- 修改的上下文
- 相关的函数和类

### 步骤 3: 逐项检查

按照检查清单，对每个修改进行：
- 静态分析（查看代码逻辑）
- 一致性验证（命名、风格）
- 完整性确认（没有一半的情况）

### 步骤 4: 输出检查结果

按以下格式输出结果：

```
## 代码质量检查结果

### ✅ 通过的检查
- 无明显 bug
- 代码风格一致
- ...

### ⚠️ 发现的问题
1. **[严重性]** 问题描述
   - 文件: path/to/file.py:123
   - 建议: 修复建议

2. **[严重性]** 问题描述
   - 文件: path/to/file.py:456
   - 建议: 修复建议

### 📝 待办事项
- [ ] 需要添加单元测试
- [ ] 需要更新文档
- ...
```

## 严重性等级

- **Critical**: 必须立即修复（如明显的 bug、安全漏洞）
- **Major**: 应该修复（如重复代码、逻辑问题）
- **Minor**: 建议修复（如风格问题、未使用变量）
- **Info**: 信息提示（如可能的优化点）
